# Completion Checklist

Before completing a task, verify:

- code is in the correct module
- layer boundaries are respected
- business logic is not in controllers or UI
- external input is validated
- authorization is enforced on the server
- decimal values are handled safely
- inventory changes use the ledger
- database transactions are used where required
- errors use stable codes
- tests cover important success and failure cases
- relevant tests pass
- type checking passes
- lint passes
- documentation is updated
- no unrelated files changed
- no secrets were added
- no temporary logging remains
