# Naming Rules

Use names that describe business intent.

Good:

- `receiveMaterial`
- `calculateAvailableStock`
- `approveStockAdjustment`
- `InsufficientStockError`

Avoid:

- `processData`
- `handleThing`
- `CommonService`
- `HelperService`
- `Utils`
- `Manager`

Boolean names should start with `is`, `has`, `can`, or `should`.

Use domain terms consistently across code, database, API, and documentation.
