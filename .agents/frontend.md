# Frontend Rules

- Keep React components focused on display and interaction.
- Do not place business rules in components.
- Do not duplicate backend authorization logic.
- Use Server Components for initial data when appropriate.
- Use Client Components only for interactive behavior.
- Show loading, empty, and error states.
- Reuse validation schemas when appropriate.
- Keep page files thin.
- Use accessible components and labels.
- Do not access the database from the frontend.

Frontend validation improves usability but never replaces backend validation.
