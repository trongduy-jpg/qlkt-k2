# TypeScript Rules

- Keep strict mode enabled.
- Never use `any`.
- Use `unknown` for untrusted values.
- Validate runtime input.
- Avoid unsafe casts.
- Avoid non-null assertions unless justified.
- Use explicit return types for exported functions.
- Prefer discriminated unions.
- Prefer readonly values where practical.
- Use exhaustive switch statements.

Do not weaken a type to hide a design problem.
